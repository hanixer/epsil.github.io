Running Bash scripts on Windows

2016-04-20

The following setup allows you to run Bash scripts from Windows’ command prompt. It also augments the command prompt with common Unix commands like ls, mv and rm, which is useful if these are ingrained into your muscle memory.

(For a list of standalone Bash shells for Windows, see alternatives.)

Usage

The minimum requirement is to install MSYS. You can then open a command prompt window and run a Bash script as follows:

sh script.sh

If the script accepts parameters, just provide them at the end:

sh script.sh --param1 val1 --param2 val2

Below follow installation instructions for MSYS, MinGW and GnuWin32.

MSYS

Install MSYS. Ensure that sh.exe is globally available by appending MSYS’ bin/ directory to the PATH environment variable.

MinGW

For more Unix utilities, augment the above setup with a complete installation of MinGW. Append its directory to PATH after MSYS.

GnuWin32

For even more Unix utilities, install GnuWin32. Append its directory to PATH after MSYS and MinGW.

Alias open

Windows does not have the open command. To alias it to start, create a C:\Windows\open.bat file containing the following:

@echo off
start %*

Alternatives

Windows 10 includes its own Linux Bash Shell, based on Ubuntu. Note that this is a standalone shell, and does not interfere with Windows’ command prompt.

Another comprehensive standalone shell is provided by the Cygwin project, which can be used on older Windows versions. MSYS and MinGW also include standalone shells, and Git for Windows provides a “Git Bash” shell.
